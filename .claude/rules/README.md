# Rules — 詳細ルールファイル

CLAUDE.mdから分離した詳細ルールを置く場所。
Claude Codeの遅延読み込みにより、該当ディレクトリにアクセスしたときだけ自動的に読み込まれる。

## CLAUDE.mdと分ける理由

CLAUDE.mdはすべてのセッションで常にコンテキストウィンドウに入る。
詳細なルールをすべてCLAUDE.mdに書くとウィンドウを圧迫して精度が落ちる。
→ 頻繁に参照しないルールはここに分離して「必要なときだけ読む」設計にする。

## ファイルの作り方

YAMLフロントマターで読み込むパスを絞れる。

```markdown
---
description: このルールの説明（1行）
paths:
  - "project/**"   # project/配下にアクセスしたときだけ読む
---

# ルール名

ルールの内容...
```

`paths` を省略するとすべてのコンテキストで読み込まれる。

## このフォルダに入っているもの

| ファイル | 内容 |
|---|---|
| `writing-style.md` | 文体・ライティング方針 |
| `characters.md` | キャラクターモード定義 |

使うほど自分の作業スタイルに合ったルールが増えていく。

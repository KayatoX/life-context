# My IDE Workspace

ライフログとLLMを連携させ、使うほど自分に最適化されていくドキュメント管理環境のスターターテンプレート。

## 原則

- **常に日本語で回答する**（コメント、ドキュメント、メッセージ含む）
- 既存ファイル編集前は必ず内容を確認する
- 破壊的な操作は事前にユーザー確認を得る
- **ファイル名は英数字・ハイフン・アンダースコア**（スキルはkebab-case統一）

## プロジェクト構造

```
[workspace-name]/
├── README.md                    # 人間向け説明
├── CLAUDE.md                    # AIへの指示書（コア）
├── .claude/
│   ├── rules/                   # 詳細ルール（遅延読み込み）
│   └── skills/                  # スキル定義
├── assistant/                   # エージェント自律管理領域
│   ├── user-insights.md         #   学習したユーザー傾向
│   └── log/                     #   操作ログ
├── user-context/                # ユーザーが自分で追加するコンテキスト
│   ├── profile.md               #   プロフィール
│   └── mindset.md               #   価値観・思考
├── scrapbook/                   # 汎用調べごと・インプット記録
│   ├── daily/YYYY-MM-DD/        #   テーマなし（日別）
│   └── theme/[theme-name]/      #   テーマ別に蓄積
├── project/                     # 特定スコープでエージェントを動かすディレクトリ
└── memo.md                      # ユーザーメモ
```

## コンテキスト管理

### 役割の分離
- `user-context/` — **ユーザーが書く場所**。プロフィール・価値観など、LLMに薄められていない一次情報を置く
- `assistant/` — **エージェントが管理する場所**。学習したインサイトや操作ログをエージェントが自律更新する

### LLMに依存しない文脈の蓄積
ルール・設計方針はgit管理されたテキストファイルに残る。特定のLLMモデルやセッションに閉じず、別のLLMに乗り換えても育てた環境をそのまま引き継げる。

## CLAUDE.md運用ルール

- **目標行数: 100行以内**（超えるとパフォーマンス低下）
- 詳細ルールは `.claude/rules/` に分離する（アクセス時に遅延読み込み）
- 情報が増えたらRulesまたはSkillsに逃がすこと
- **構成原則**: WHAT（何のためか）/ WHY（なぜこの方針か）/ HOW（どう動かすか）
